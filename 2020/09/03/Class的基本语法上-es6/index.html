Class的基本语法上(es6)

发表于 2020-09-03 更新于 2020-09-08 分类于 ES6入门
本文字数： 1.9k 阅读时长 ≈ 2 分钟

ES6总结系列之 class(上) 篇

1.何为class

  1. ES6 的 class 完全可以看作构造函数的另一种写法，使用的时候，也是直接对类使用new命令，跟构造函数的用法完全一致
  2. class 也可以看作只是一个语法糖，它的绝大部分功能，ES5 都可以做到，新的class写法只是让对象原型的写法更加清晰、更像面向对象编程的语法而已

2.基本用法

我们先来看一个例子

  1. this关键字代表实例对象

  2. constructor为构造方法, 通过 new 命令生成实例时自动被调用， 没有显式定义时一个空的 constructor 会被默认添加, constructor 方法默认返回实例对象（即this）

  3. 定义类方法不需要关键字function

  4. 方法之间不需要逗号分隔，加了会报错

  5. 类的所有方法都定义在类的prototype属性上面

  1. prototype 对象的 constructor 属性，直接指向“类”的本身，这与 ES5 的行为是一致的
  2. 类的内部所有定义的方法，都是不可枚举的
  1. class 必须使用 new 调用，否则会报错。这是它跟普通构造函数的一个主要区别，后者不用 new 也可以执行

注意点

  1. 类和模块内部，默认就是严格模式
  2. 类不存在变量提升
  1. name 属性返回 class 关健字后面的类名
  1. 如果某个方法之前加上星号（*），就表示该方法是一个 Generator 函数

总结

基本用法

  1. this关键字代表实例对象
  2. constructor为构造方法, 通过 new 命令生成实例时自动被调用， 没有显式定义时一个空的 constructor 会被默认添加, constructor 方法默认返回实例对象（即this）
  3. 定义类方法不需要关键字function
  4. 方法之间不需要逗号分隔，加了会报错
  5. 类的所有方法都定义在类的prototype属性上面
  6. prototype 对象的 constructor 属性，直接指向“类”的本身，这与 ES5 的行为是一致的
  7. 类的内部所有定义的方法，都是不可枚举的
  8. class 必须使用 new 调用，否则会报错。这是它跟普通构造函数的一个主要区别，后者不用 new 也可以执行

    注意要点

  9. 类和模块内部，默认就是严格模式
  10. 类不存在变量提升
  11. name 属性返回 class 关健字后面的类名
  12. 如果某个方法之前加上星号（*），就表示该方法是一个 Generator 函数
